以下是涵蓋 DIC3-1 及其後續計畫 DIC3-2 的明確專案說明：

專案說明：物聯網資料記錄 (DIC3-1 & DIC3-2)
DIC3-1：PHP 與 MySQL 實作
專案的第一階段使用 PHP 與 MySQL 實作了一個小型的物聯網資料記錄系統。

一個名為 `addData.php` 的 PHP 腳本透過 HTTP GET 請求接收感測器讀數，例如：

http://localhost/aiot/addData.php?temp=11&humid=11

該腳本驗證輸入數值（`temp` 和 `humid`）並將其插入 MySQL 資料庫的 `sensors` 資料表中。

`sensors` 資料表通常包含以下欄位：

id (INT, 主鍵, 自動遞增)

temp (FLOAT)

humid (FLOAT)

created_at (TIMESTAMP, 預設 CURRENT_TIMESTAMP)

這套架構允許物聯網裝置或網頁客戶端輕鬆地將環境資料傳送至伺服器，以便儲存和後續分析。

DIC3-2：Flask 與 SQLite 移轉
專案的第二階段將後端從 PHP/MySQL 升級為 Python Flask 搭配 SQLite3，使系統更輕量且具備更高的可攜性。

PHP 腳本將替換為 `app.py`，這是一個處理 HTTP GET 請求以記錄感測器資料的 Flask 應用程式。

MySQL 資料庫由一個 SQLite3 資料庫檔案取代：`aiotdb.db`。

`sensors` 資料表定義如下：

```sql
CREATE TABLE sensors (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    temp FLOAT DEFAULT 0,
    humid FLOAT DEFAULT 0,
    time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

Flask 應用程式處理如下的請求：

http://localhost:5000/addData?temp=11&humid=11

並在 SQLite 資料庫中插入或更新資料。

此次移轉提升了跨平台相容性、簡化了部署流程，並減少了伺服器的資源使用。

總結：
這個小專案展示了完整的物聯網資料收集工作流程：透過 HTTP 接收感測器資料、進行驗證，並儲存至資料庫中。DIC3-1 使用 PHP/MySQL，而 DIC3-2 則將後端現代化為 Flask/SQLite，以提供更具可攜性和易於維護的解決方案。
